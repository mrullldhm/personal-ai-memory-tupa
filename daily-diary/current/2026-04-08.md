## Main Topics Discussed

1. **New export column layout from Z**: Redesigned export format for all 3 types — summary, daily, hourly. New columns: device manufacturer ratios (top 5), OS ratios (iOS/Android/Other), staytime (`stay_p50`). Replaced old 4-query-per-branch approach.

2. **SQL-first architecture**: Single SQL file per export type (`export-summary.sql`, `export-daily.sql`, `export-hourly.sql`) in `src/sqls/`. Each runs one query against `emerald.analytics_daily` or `emerald.analytics_hourly`.

3. **BigQuery reserved keyword bug**: `AS new` and `AS returning` caused query failures — both are reserved BigQuery keywords. Renamed to `new_count` and `returning_count`. Key lesson: BigQuery stricter than standard SQL on reserved words.

4. **SQL-to-dist copy requirement**: `tsc --watch` does NOT copy `.sql` files from `src/sqls/` to `dist/sqls/`. Must do manually after every SQL edit, or via `pnpm build`.

5. **Ratio storage convention**: All device and OS ratios stored as raw 4-decimal values (e.g. `0.4231`). Client/frontend multiplies by 100 for percentages.

6. **`stay_p50` explained**: 50th percentile (median) of visitor stay time, in minutes.
